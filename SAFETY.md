# SAFETY.md · Art_elect — Архитектура безопасности

> Обеспечение информационной безопасности ИСУ РЭС в соответствии с ФЗ-187 (КИИ).

---

## Нормативная база

- **ФЗ-187** «О безопасности объектов критической информационной инфраструктуры РФ»
- **Приказ ФСТЭК №239** — требования к системам значимых объектов КИИ
- **ГОСТ Р 56545-2015** — уязвимости ИС
- **ПТЭЭП** — правила технической эксплуатации электроустановок

## Архитектура защищённого периметра

```
[Полевой уровень] → [Сетевой уровень (МЭ)] → [Корпоративный уровень (VPN)]
     |                      |                           |
  RTU/датчики         Межсетевые экраны          Серверы/БД/ML
  Шифрование          IDS/IPS                    Многофакторная Аут.
```

### Уровни защиты

1. **Периметр** — межсетевые экраны между уровнями, IDS/IPS на сетевом уровне
2. **Каналы** — TLS 1.3 для всех внешних соединений, VPN для удалённого доступа
3. **Доступ** — многофакторная аутентификация (пароль + токен/биометрия)
4. **Аудит** — логирование всех действий, журнал событий (срок хранения — 1 год)

## Механизмы аутентификации, авторизации и аудита

### Аутентификация
- Интеграция с LDAP/AD «Башкирэнерго»
- Многофакторная: пароль + OTP-токен (TOTP)
- Аппаратные токены для критических операций

### Авторизация (RBAC)
| Роль | Доступ |
|------|--------|
| Диспетчер | Просмотр + подтверждение рекомендаций |
| Инженер АСУ | Настройка моделей, обновление правил |
| Администратор | Управление пользователями, аудит |
| Наблюдатель | Только просмотр |

### Аудит
- Фиксация всех управляющих команд и рекомендаций
- Журнал доступа к моделям и данным
- Ежеквартальные аудиты соответствия ФЗ-187

## Защита данных и каналов связи

- **Телеметрия**: шифрование на уровне протокола (MQTT over TLS)
- **Управляющие команды**: подписание ЭЦП
- **Хранение**: шифрование БД (PostgreSQL TDE), ротация ключей каждые 90 дней
- **Резервирование**: репликация данных, холодный резервный ЦОД

## Автоматизированное управление (безопасность по умолчанию)

Система работает в полностью автоматическом режиме:
1. Модуль мониторинга собирает телеметрию
2. Модуль диагностики классифицирует состояние сети
3. Модуль прогнозирования (CatBoost) формирует прогноз
4. Экспертная система проверяет по нормативным ограничениям (ПТЭЭП, ГОСТ)
5. **Управляющее воздействие применяется автоматически**
6. Фиксация действия в журнале аудита

### Гарантии безопасности
- Автоматическая верификация всех воздействий перед исполнением
- Приоритет нормативных ограничений над алгоритмическими рекомендациями
- Полный аудит всех действий с возможностью отката
- Аварийный останов (Emergency Shutdown) при выходе параметров за критические пределы
- Дублирование критических каналов управления
