# AGENTS.md · Art_elect — Правила для агентов

> Правила работы с git: коммиты, пуш, CI/CD.

---

## Протокол коммита (обязательно)

Перед каждым коммитом:

1. `git status` — проверить статус
2. `git diff --staged` — проверить, что попадает в коммит
3. Проверить, что сообщение коммита содержит:
   - префикс-эмодзи и тип (`[DOC] feat:`, `[EXP] feat:`, `[FIX] fix:`)
   - краткое описание в заголовке
   - ключевые метрики/результаты в теле
   - инсайты (если есть)

## Авто-коммиты GitHub Actions

При `git push` возможен авто-коммит бота с обновлением навигации.
При ошибке `rejected (fetch first)` — использовать:

```bash
git pull --rebase --autostash
git push
```

**Никогда** не делать `git push --force`.

## Правило содержательных коммитов

Каждый коммит — самодостаточный документ:

```
<emoji> <тип>: <что сделано — кратко, но информативно>

<основной результат>:
- Ключевые метрики (MAE, RMSE, R², BA)
- Какая проблема решена
- Важные цифры и сравнения
- Инсайты/открытия
```

**Плохо:** `[FIX] fix: исправлена ошибка`  
**Хорошо:** `[FIX] fix: исправлен leakage — train_test_split shuffle=False`
