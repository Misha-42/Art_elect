## 1. Общие правила

- Все changelog entries, git reports, review notes — на русском языке.
- English words — только с переводом в скобках при первом упоминании.
- При каждом обращении к файлу указывать его полный путь относительно корня.
- После завершения крупной задачи (коммит + пуш) предлагать 2–3 варианта дальнейших действий.

## 2. Два режима работы

### Режим A: Автономный (есть прямое задание)
- Не задавать уточняющих вопросов — делать.
- Довести до конца: код → запуск → проверка → коммит → пуш.
- После завершения — предложить 2–3 варианта дальнейших действий.

### Режим B: Совместный (нет прямого задания)
- Предлагать варианты решений по принципу Парето 80/20.
- Если видишь два способа — показать оба с трейдоффами.
- Краткость: сначала суть (1–3 предложения), детали по запросу.

## 3. Воспроизводимость (ML)

## 5. Безопасность (ФЗ-187)

- Все изменения, затрагивающие безопасность, маркировать префиксом `[SEC]`.
- Не предлагать отключение механизмов аутентификации/авторизации.
- Управляющие воздействия — ТОЛЬКО в режиме «советчика» (подтверждение диспетчером).

## 6. Структура коммитов


## 7. Чего не делать

- Не менять структуру папок без явного запроса.
- Не добавлять зависимости без согласования.
- Не удалять архивные эксперименты.
- Не переходить на другой проект без команды.
