КЛЮЧЕВЫЕ ТЕХНИЧЕСКИЕ РЕШЕНИЯ (KEY TECHNICAL DECISIONS)
=====================================================
Проект: Art_elect — ИСУ Ишимбайскими РЭС
Дата: 10.06.2026

---

1. ГИБРИДНАЯ АРХИТЕКТУРА
   Решение: ML (CatBoost) + Экспертная система
   Обоснование: вероятностные прогнозы + детерминированные правила
   Статус: утверждено

2. ТРЁХУРОВНЕВАЯ АРХИТЕКТУРА
   Решение: полевой / сетевой / корпоративный уровни
   Обоснование: масштабируемость, отказоустойчивость, совместимость с SCADA
   Статус: утверждено

3. РЕЖИМ РАБОТЫ «СОВЕТЧИК»
   Решение: рекомендации диспетчеру, не автоматическое управление
   Обоснование: требования ФЗ-187 (КИИ), безопасность
   Статус: утверждено

4. ПРИОРИТЕТ НОРМАТИВНЫХ ОГРАНИЧЕНИЙ
   Решение: нормативные правила (ПТЭЭП, ГОСТ) имеют приоритет над ML
   Обоснование: безопасность эксплуатации электроустановок
   Статус: утверждено

5. ПРОГНОЗИРОВАНИЕ CATBOOST
   Решение: градиентный бустинг как основной метод прогнозирования
   Обоснование: лучшая работа с пропусками и категориальными признаками
   Статус: утверждено

6. ХРАНЕНИЕ ВРЕМЕННЫХ РЯДОВ
   Решение: TimescaleDB/InfluxDB для телеметрии, PostgreSQL для метаданных
   Обоснование: специализированные СУБД для временных рядов
   Статус: утверждено

7. ПРОТОКОЛЫ ОБМЕНА
   Решение: MQTT (телеметрия), REST/gRPC (управление), OPC UA (SCADA)
   Форматы: JSON (отладка), Protobuf (высоконагруженные каналы)
   Статус: утверждено

8. АВТОМАТИЗИРОВАННОЕ УПРАВЛЕНИЕ (АКТУАЛИЗАЦИЯ)
   Решение: система работает в полностью автоматическом режиме, без режима «советчика»
   Обоснование: управляющие воздействия применяются автоматически после верификации
   Изменение: режим «советчика» удалён из архитектуры (был в НИР-2, но актуализирован)
   Статус: утверждено

9. ИНФРАСТРУКТУРА
   Решение: Docker + Kubernetes, контейнеризация модулей
   Ресурсы: от 16 vCPU / 64 ГБ ОЗУ
   Статус: утверждено

10. СИНТЕТИЧЕСКИЕ ДАННЫЕ
   Решение: на этапе разработки используем синтетическую телеметрию
   Обоснование: ускорение работы, отсутствие доступа к реальной SCADA на старте
   Генератор: experiments/energy-ml-v1_baseline/code/generate_synthetic_data.py
   Параметры: 10000+ строк, 3 режима (норма/предупреждение/авария), автокорреляция, сезонность
   Статус: утверждено
